Programmer's Lifehacks: A Curated Cheat Sheet for Developers

Sep 2023 • 10 min read •
DevOps Docker Kubernetes

Introduction

Over the years of working as a software engineer, I have accumulated a collection of commands, configurations, and tips that I use on a daily basis. Instead of keeping them scattered across sticky notes and random text files, I decided to organize everything into a single public repository.

The full collection is available on GitHub: devalex-tsi/programmers-lifehacks. It covers Docker, Kubernetes, Helm, Kafka, PostgreSQL, Git, networking, Linux, VS Code settings, SEO tools, and even smart AI prompts. No fluff — just practical stuff you will actually use.

In this article, I will walk through the key sections of the repository and highlight the most useful commands and tricks from each category.

Docker

The Docker section covers the entire container lifecycle: running, inspecting, building, and debugging. Here are some of the most frequently used commands from the COMMANDS DOCKER.txt file: 

# List running containers
docker ps

# List all containers (including stopped)
docker ps -a

# Run a PostgreSQL container with custom settings
docker run --name pg-doc -p 8165:5432 \
  -e POSTGRES_USER=docker \
  -e POSTGRES_PASSWORD=*** \
  -e POSTGRES_DB=pg-doc \
  --network=bridge -d postgres

# Inspect container network settings
docker inspect core-doculab-backend \
  -f "{{json .NetworkSettings.Networks }}"

# List containers with their networks
docker ps -a --format '{{ .ID }} | {{ .Names }} | {{ json .Networks }}'

The file also includes commands for building and tagging images, executing commands inside running containers, and publishing .NET applications into Docker images: 

# Build an image from Dockerfile
docker build -t counter-image -f Dockerfile .

# Run the image and map ports
docker run -p 5203:80 counter-image

# Tag an image for a registry
docker tag date-time-image:latest alex/date-time-image:latest

# Execute a shell inside a running container
docker exec -it core-doc-backend bash

Kubernetes

The Kubernetes section is the largest in the repository. It covers everything from Minikube setup to real-time log tailing in production clusters. The full reference is in COMMANDS KUBER.txt.

Cluster setup and context switching: 

# Start Minikube with VirtualBox driver
minikube start --driver='virtualbox'

# Add a worker node
minikube node add --worker

# Switch between cluster contexts
kubectl config current-context
kubectl config use-context corecluster

Pod management and debugging: 

# Get all pods across namespaces
kubectl get po -A

# Describe a specific pod
kubectl describe pods pod-name

# Execute a command inside a pod
kubectl exec my-pod -- date

# Port-forward to access a pod locally
kubectl port-forward my-pod 7788:80

Deployments and scaling: 

# Create a deployment
kubectl create deploy my-deployment --image alex/date-time-image

# Scale to 3 replicas
kubectl scale deploy my-deployment --replicas 3

# Set up autoscaling
kubectl autoscale deploy my-deployment \
  --min=4 --max=6 --cpu-percent=80

# Check rollout status and history
kubectl rollout status deploy/my-deployment
kubectl rollout history deploy/my-deployment

Log viewing: One thing I find myself doing constantly is checking pod logs. The repository includes several patterns for this: 

# Last 100 lines
kubectl logs <pod-name> -n <namespace> --tail=100

# Scroll through logs with 'more'
kubectl logs <pod-name> -n <namespace> | more

# Real-time log streaming
kubectl logs -f <pod-name> -n <namespace>

Helm

Helm chart management is covered in COMMANDS HELM.txt. It includes the essential commands for listing, installing, and configuring Helm releases: 

# List installed releases
helm list

# Search available charts
helm search repo

# Install a release with custom values
helm install core-doc-db bitnami/postgresql \
  --set primary.persistence.existingClaim=core-doc-db-pvc \
  --set auth.postgresPassword=*** \
  --set auth.username=docapp \
  --set auth.password=*** \
  --set auth.database=doc

This is particularly useful for quickly spinning up databases and other infrastructure components in Kubernetes with custom persistence and authentication settings.

Databases & Kafka

The repository includes commands for PostgreSQL connection and user management ( COMMANDS POSTGRES.txt), as well as Kafka topic creation via Confluent CLI ( COMMANDS KAFKA.txt).

PostgreSQL basics: 

# Connect to PostgreSQL
psql -h $(hostname -i) -U docapp

# List users
\du

# Change a user's password
ALTER USER yourusername WITH PASSWORD 'yournewpass';

A particularly handy trick is extracting PostgreSQL secrets from Kubernetes: 

# Get admin password from Kubernetes secret
export POSTGRES_ADMIN_PASSWORD=$(kubectl get secret \
  --namespace annset core-doc-db-postgresql \
  -o jsonpath="{.data.postgres-password}" | base64 -d)

# Get application password
export POSTGRES_PASSWORD=$(kubectl get secret \
  --namespace annset core-doc-db-postgresql \
  -o jsonpath="{.data.password}" | base64 -d)

Kafka topic creation with schema validation via Confluent CLI: 

confluent kafka topic create requirement-hide \
  --url localhost:8082 \
  --no-auth \
  --if-not-exists \
  --replication-factor 1 \
  --config confluent.key.schema.validation=true,\
confluent.value.schema.validation=true

Git

The Git section ( git.txt) contains a useful trick for bulk-changing the author name and email across multiple commits. This is something you might need when you accidentally commit with the wrong identity: 

# Change author and email in the last 7 commits:
# 1. Set user.name and user.email locally
git config user.name "Your Name"
git config user.email "your@email.com"

# 2. Rebase and amend each commit
git rebase --onto HEAD~7 \
  --exec "git commit --amend --reset-author --no-edit" HEAD~7

# 3. Force push (with safety)
git push --force-with-lease

Be careful with force pushing — always use --force-with-lease instead of --force to avoid overwriting other people's work.

Networking

The NETWORK folder in the repository covers DNS diagnostics and SSL/TLS tools. This includes:

  • DNS diagnostics — nslookup commands for troubleshooting domain resolution issues.
  • Wireshark — packet analysis for debugging network issues at the protocol level.
  • OpenSSL — certificate inspection and SSL/TLS debugging.
  • SSL Labs — deep analysis of server SSL/TLS configuration and security grading.
  • GeoCerts SSL Tools — comprehensive SSL diagnostics for verifying certificate chains.

These tools are essential when debugging connectivity issues in distributed systems or verifying SSL certificates in production environments.

Developer Tools

The repository also includes sections that go beyond pure DevOps:

VS Code Settings ( Settings VS Code.txt): recommended extensions like Image Preview, Live Server, and Auto Rename Tag, plus productivity tips like enabling Auto Save on Focus Change.

SEO Toolkit ( SEO folder): a complete SEO starter kit with tools like Screaming Frog, Sitebulb, Google Search Console, and Lighthouse. It also covers keyword research tools (Google Trends, Answer The Public, Also Asked) and Google search operators (site:, intitle:, inurl:).

Smart AI Prompts ( smart-ai-prompts folder): ready-to-use prompt templates for code review (C#/.NET), Azure DevOps + AWS deployment automation, and translation tasks with tone control. These are practical prompts that you can copy and adapt for your own workflow.

Linux / VirtualBox ( linux folder): step-by-step guide for setting up VirtualBox Guest Additions on Ubuntu, enabling bidirectional clipboard and drag-and-drop between Windows host and Ubuntu guest.

Conclusion

This repository is a living document — I keep adding new commands and tools as I discover them in my daily work. Whether you are a backend developer, a DevOps engineer, or someone who just wants to have a reliable cheat sheet at hand, I hope you find it useful.

Feel free to explore the full repository, star it for quick access, and contribute your own lifehacks: github.com/devalex-tsi/programmers-lifehacks.
